00 网站图、BFS 与 crawler 状态
学习

Crawler 把页面看作节点、把链接看作边。先从顺序版本开始，队列顺序是可观察且确定的。

  • urls_to_crawl 保存已经发现、等待抓取的工作。
  • discovered_urls 阻止环路和重复 URL。
  • fetched_urls 记录完成的页面。

01 Futures、Tasks 与等待进展
比较

线程池在线程中运行阻塞抓取函数并返回 Future 句柄。Asyncio 调度会在 await 时让出控制权的 Task。

两个版本都会限制正在执行的工作量，并在第一个操作完成时继续推进。

运行
python3 compare_examples.py

# 找出对应思想：
wait(..., FIRST_COMPLETED)
asyncio.wait(..., FIRST_COMPLETED)

自查：为什么并发完成顺序可能不同于顺序 BFS，即使所有可达 URL 最终仍会被找到？

02 Timeout：限制等待占用容量的时间
学习

非阻塞等待仍会占用一个请求槽位。Timeout 防止某个很慢的页面无限期占住容量。

动手
python3 -m lessons.lesson_timeout

return await asyncio.wait_for(
    get_html_content_async(url),
    timeout=timeout_seconds,
)

实验：将 timeout 调到低于模拟页面延迟。Crawler 应记录什么结果？

03 Retry with backoff：恢复但不过度请求
学习

只对暂时性错误进行 retry。指数退避会拉开后续尝试的距离，让有压力的服务有恢复时间。

动手
python3 -m lessons.lesson_retry

delay_seconds = base_delay_seconds * (
    2 ** (attempt_number - 1)
)

自查：为什么错误 URL 或权限失败通常不应该 retry？

04 Semaphore：限制活跃异步请求
学习

Semaphore 限制受保护抓取区域中的活跃工作。它能避免请求突发，但不会限制已经发现的 URL 总量。

动手
python3 -m lessons.lesson_semaphore

async with semaphore:
    await fetch_page()

自查：还需要什么单独的限制，才能避免巨大网站不断扩大抓取 frontier？

05 Cancellation：停止并收集未完成 tasks
学习

Cancellation 发出停止工作的信号。继续 await 被取消的 tasks 可以让清理完成，并观察它们的最终状态。

动手
python3 -m lessons.lesson_cancellation

task.cancel()
await asyncio.gather(
    task, return_exceptions=True
)

自查：为什么创建 tasks 却没有最终 await 或 cancel 它们会有风险？

共享边界：同一个离线网站提供阻塞 fetch 和 async fetch，因此对比是公平的。
def get_html_content(url: str) -> str:
    time.sleep(PAGE_DOWNLOAD_DELAYS.get(url, 0.02))
    return _read_example_page(url)

async def get_html_content_async(url: str) -> str:
    await asyncio.sleep(PAGE_DOWNLOAD_DELAYS.get(url, 0.02))
    return _read_example_page(url)
Threads：submit() 返回 Future；wait() 让协调逻辑等待至少一次抓取完成。
future = thread_pool.submit(self.crawl_page, url)
self.pending_futures.add(future)

done_futures, pending_futures = wait(
    self.pending_futures,
    return_when=FIRST_COMPLETED,
)
for future in done_futures:
    future.result()
Asyncio：tasks 在 event loop 中推进，并在模拟请求等待期间让出控制权。
task = asyncio.create_task(self.fetch_links(url))
pending_tasks.add(task)

done_tasks, pending_tasks = await asyncio.wait(
    pending_tasks,
    return_when=asyncio.FIRST_COMPLETED,
)
for task in done_tasks:
    url, discovered_links = task.result()
